# Paper2 Extension — Implementation Ledger

Running log of decisions, measured values, and open items for the paper2 (Kyung et al., IEEE CAL 2026) replication extension. Written during implementation; feeds the final deviations writeup.

## Decisions (grounded, adversarially reviewed pre-implementation)
- Preset capacities in GiB (codebase convention); lifetime TBW uses decimal 5×512e9 B/device in Python (flash-vendor convention, paper Eq. 2). GB-vs-GiB sensitivity ≈7% — documented, not tunable.
- paper2_mode zeroes the ENTIRE exposed page-latency term (fill + residual) in both read paths and both optimizer mirrors.
- SRAM gate bypassed via `unbounded_sram_gate` bool (+infinity limit), not a magic capacity.
- Fig4 write population = admission (prefill-KV landing) + per-step decode appends. Eq.(1)'s "−1 token" is ≤0.012% — documented, not modeled.
- C2C read composition default SUM (serial); MAX behind `c2c_read_composition`. To be confirmed against Fig5 NVLink6 TPOTs.
- V_weight for batch/TBW/ratio math = simulator's own mapping accounting (duplicated non-expert × DP + distributed routed experts), emitted as P2_WEIGHT_BYTES_NODE. No hand-picked constants.
- disable_irope = attn_chunk_size→0 after preset selection (all consumers fall through the existing ==0 full-attention path). Grounding: paper2 §II "standard full attention"; Fig5 baseline 32K≈¼×8K at equal TPOT.
- Size-biased initial fill (accept ∝ Lout) for paper2 workloads; refills unbiased at age 0 — stationary distribution of continuous batching (inspection paradox).
